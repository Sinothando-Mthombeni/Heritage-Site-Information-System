# Phase 4 – MongoDB (NoSQL)

## Objective
Introduce MongoDB for semi-structured and read-optimized data that benefits
from document-based storage.

## Use Cases
- Visitor activity logs
- Site popularity metrics
- Flexible metadata

## Data Model
- Embedded documents
- Denormalized structures
- Read-optimized schemas

## Operations Implemented
- Collection creation
- Data seeding
- Aggregation pipelines
- Indexing strategies

## Why MongoDB
- Schema flexibility
- Fast read access
- Natural fit for analytical summaries

## Outcome
MongoDB complements PostgreSQL without replacing it.
