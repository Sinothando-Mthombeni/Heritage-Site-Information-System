# Phase 3.5 – Reporting Layer

## Objective
Demonstrate advanced SQL querying by implementing reporting-focused queries
separate from transactional logic.

## Reporting Focus
- Aggregated statistics
- Historical trends
- Read-optimized queries

## Techniques Used
- CTEs for clarity
- Window functions for ranking
- Materialized-style queries (logical)

## Why SQL for Reporting
Relational databases excel at structured aggregations and historical analysis,
making PostgreSQL ideal for reporting use cases.

## Outcome
A clean separation between operational data and analytical queries.
